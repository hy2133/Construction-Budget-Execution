# Orchestration Guidelines (작업 지침)

이 파일은 이 프로젝트의 모든 코드 작업 및 파일 관리에 대한 **최상위 지침(Supreme Authority)**입니다.

## 1. VBA 파일 인코딩 및 처리 규칙 (수정본)

사용자의 편집 환경(VS Code 등)에서 한글이 정상적으로 보이도록 하기 위해 다음 규칙을 준수합니다.

### [인코딩 표준]
- 모든 `.bas` 파일은 **UTF-8 (BOM 없음)** 인코딩으로 저장합니다.
- 줄바꿈(Line Endings)은 **CRLF (`\r\n`)** 형식을 권장합니다.

### [파일 수정 및 저장 방법]
- `write_to_file` 툴을 사용하면 기본적으로 UTF-8로 저장됩니다. 
- 파워쉘을 사용할 경우에도 유니코드가 보존되도록 주의해야 합니다.

## 2. 코드 최적화 지침 (Karpathy Guidelines 준수)

- `Select` 및 `Selection` 사용을 최소화하고 시트/범위를 직접 참조합니다.
- 불필요한 추상화나 기능을 추가하지 않고, 요청받은 기능만 간결하게 구현합니다.
- 기존 스타일을 존중하며 국소적인(Surgical) 변경만 수행합니다.

## 3. 커뮤니케이션 규칙

- 모든 설명과 주석은 **한국어**로 작성합니다.
- [결론/해결책] -> [코드] -> [상세 설명] 순서의 두괄식 답변을 지향합니다.
- 잠재적 리스크나 대안이 있을 경우 적극적으로 제안합니다.
